## 3. 브랜드 톤 & 문체

| 항목 | 규칙 |
|------|------|
| 어조 | 존댓말 (~합니다, ~하세요) 통일 |
| 스타일 | 뉴스 형식 — 객관적 사실 전달, 주관 최소화 |
| 페르소나 | "정보를 전달하는 전문가" — 추천이 아닌 정리 |
| 1인칭 | "저는/제가" 사용 금지 (전문성 훼손). 수동/명사형으로 |
| 2인칭 | "여러분", "당신" 사용 금지. "~하시는 분", "크리에이터" 등 간접 지칭 |
| 문장 길이 | 평균 15~20자. 한 호흡에 읽히게 |
| 전문용어 | 첫 언급 시 괄호 설명 필수 (예: "E-E-A-T(경험·전문성·권위·신뢰성)") |
| 감정어 | 과장 표현 금지 ("반드시", "100%", "대박", "무조건") |

### 좋은 예시
> "YouTube는 2026년 4월 20일, 새로운 알고리즘 업데이트를 공식 발표했습니다. 핵심은 '시청 유지율'에서 '완시청 비율'로의 평가 기준 전환입니다."

### 나쁜 예시
> "여러분 대박 뉴스가 있어요! 유튜브 알고리즘이 바뀌었는데 이거 정말 반드시 알아야 합니다!!"

- 유튜브/틱톡/인스타그램/X 등 **SNS 플랫폼의 알고리즘·정책 변화**
- 크리에이터에게 유용한 **새로 출시된 AI 툴**
- 기존 AI 툴의 **중요 업데이트/가격 변경**
- **수익화 정책** 변화 (광고, 멤버십, 슈퍼챗 등)
- **창작자 법·세금** 관련 공식 가이드 변화
- AI × SNS 교차 주제 (예: "AI 자막 생성 도구 TOP 5", "틱톡의 AI 기능 추가")

### ❌ 제외 주제
- 검증되지 않은 **루머·추측**
- 정치·종교·사회 논란
- 개인 저격/비판
- 특정 유튜버의 개인사
- 플랫폼 **약관 위반** 관련 노하우 (예: 조회수 조작법)
- 가짜 정보/딥페이크 활용법

### 🎯 핵심 전략: "저수지 가두기"
모든 글은 **최소 1개 이상의 내부 링크**로 기존 글/다른 트렌드 글과 연결.

## 8. 출처 & 팩트체크 규칙

### 출처 등급

| 등급 | 예시 | 사용 가능 여부 |
|------|------|---------------|
| **A급** | Google/YouTube/TikTok/Meta/OpenAI/Anthropic 공식 블로그·발표 | ✅ 본문 직접 인용 |
| **B급** | TechCrunch, The Verge, Wired, MIT Tech Review, Reuters, 연합뉴스 등 | ✅ 본문 참고·인용 |
| **C급** | Reddit, X/Twitter, 개인 블로그 | ⚠️ 참고만, 본문 인용 금지 |
| **D급** | 루머 사이트, 익명 소스, 검증 안 된 매체 | ❌ 사용 금지 |

### 필수 표기
- 모든 수치/인용은 **출처 표시 필수**
- 본문 끝 `[출처]` 섹션에 A급 1개 이상 링크 필수
- "~라고 합니다" 같은 추측성 표현 대신 **"[출처]에 따르면 ~"**

### 날짜 명시 (절대 원칙)

**모든 날짜는 연도를 반드시 포함한다.** 글은 수 년간 남아있을 수 있고, 1~2년 후 읽는 독자가 "지난 4월"이 언제의 4월인지 혼동하면 안 됨.

#### 허용되는 형식
- ✅ `2026년 4월 15일` — 가장 자연스럽고 명확
- ✅ `2026-04-15` — 본문 중 날짜 강조할 때 (예: 발표일(2026-04-15))
- ✅ `2026년 4월` — 월 단위
- ✅ `2026년 상반기` — 기간 표현
- ✅ `April 15, 2026` (영문)

#### 절대 금지 표현
- ❌ `지난 4월 15일` — 어느 해의 4월?
- ❌ `이번 주`, `지난 주`, `다음 주` — 언제 기준?
- ❌ `이번 달`, `지난 달`
- ❌ `올해`, `작년`, `내년` — 미래에 읽히면 혼동
- ❌ `최근`, `요즘`, `얼마 전`, `며칠 전`
- ❌ 영어: `recently`, `last week/month`, `this year`, `a few days ago`

#### 예외 (맥락상 명확한 경우)
- `이번 발표` / `이번 업데이트` — 글 내부에서 특정 사건을 가리킬 때는 사용 가능
- `당시` / `그 후` — 앞서 명시된 날짜 이후를 지칭할 때

#### 좋은 예시
> ✅ "2026년 4월 15일, 틱톡은 Symphony AI 도구를 공개했습니다."
> ✅ "On April 15, 2026, TikTok opened Symphony..."

#### 나쁜 예시
> ❌ "지난 4월 15일, 틱톡이..."
> ❌ "얼마 전 틱톡이 새 도구를 공개..."
> ❌ "Recently, TikTok announced..."

## 10. HTML 기술 규칙

### 기존 템플릿 재사용
- 모든 트렌드 글은 **`.post-header`, `.post-title`, `.post-desc`, `.post-body`** 구조 유지
- CSS는 `../css/style.css` 그대로 사용 (새 CSS 파일 생성 금지 — 일관성)
- 네비/푸터는 기존 글과 100% 동일하게 복사

### 파일 이름 규칙
`YYYY-MM-DD-slug.html`
- 예: `2026-04-23-youtube-algorithm-update.html`
- slug는 영문 kebab-case, 3~5단어

### 폴더 구조
```
creator-jungbok/
├── trends/
│   ├── index.html (목록 페이지 — 자동 재생성)
│   ├── 2026-04-23-slug.html
│   ├── 2026-04-24-slug.html
│   └── ...
└── docs/
    ├── content-rules.md (이 파일)
    └── seo-rules-latest.md (주 1회 자동 갱신)
```

### 내부 링크 필수
- 본문에 **기존 글(/basics/, /practice/) 링크 1~2개**
- 끝 "더 읽어볼 글"에 **트렌드 내 관련 글 2~3개**
- 모든 내부 링크는 상대 경로 사용

## 11. 금지 / 필수 요소

### 🚫 금지 표현
- 과장: "반드시", "100%", "무조건", "확실히", "절대"
- 감정어: "대박", "충격", "미쳤다", "레전드"
- 클릭베이트: "~하는 X가지 이유", "아무도 모르는~"
- 추측: "~것 같다", "~일 수도", "~라고 추정"
- 1인칭: "저는", "제가", "우리"

### ✅ 필수 요소
- 글 상단: **리드 문단** (핵심 5W1H 요약)
- 글 하단: **요약 박스** (3~5줄, `.callout.tip`)
- 글 하단: **출처 섹션** (`.callout.info`, A급 출처 1개 이상)
- 글 하단: **"더 읽어볼 글"** (내부 링크 3개)
- 게시일 표시 (YYYY-MM-DD)
- 읽기 예상 시간 표시
